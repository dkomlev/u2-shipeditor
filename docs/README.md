# Документация U2 Ship Editor

Этот каталог содержит спецификации форматов, обзоры архитектуры и рабочие заметки для инструментов U2 Ship Editor, FlightTest и Pilot Assist.

Основной язык документации — русский, часть материалов на английском. Для новых файлов рекомендуется использовать кодировку UTF‑8.

## Аудит и качество кода

История аудитов и технического долга:

- Консолидированное резюме см. в `docs/AUDIT_SUMMARY.md`.
- Детализированные списки задач и замечаний — в `docs/AUDIT_RESULTS.md`.

Текущие ключевые рекомендации по качеству (кратко):

1. **Кодировка** — избегать CP1251; новые файлы и правки делать в UTF‑8, постепенно мигрируя старые документы и текст в UI.
2. **Структура модулей** — большие файлы (`js/app.js`, `js/nominals.js` и др.) по возможности декомпозировать на более мелкие модули без потери простоты сборки.
3. **Тесты** — поддерживать существующие unit‑тесты (`tests/`, `test/`), при внесении изменений в физику и адаптеры дополнять их минимальными проверками.
4. **Согласованность версий** — следить за тем, чтобы упоминания версий (`0.6.x`, `0.7.x`, `0.8.x`) в документации соответствовали актуальному состоянию схем и кода.

## Как ориентироваться в документации

Рекомендуемый маршрут чтения для нового участника:

1. `README.md` в корне репозитория — общее представление о проекте.
2. Этот файл — обзор структуры документации.
3. `docs/U2_ShipConfig_v0.6.md` — формат конфигов кораблей.

5. Документы про FlightTest / Pilot Assist в зависимости от интересующей области.

По мере развития проекта сюда можно добавлять ссылки на новые документы и краткие аннотации к ним.
